Introduction

Linux is a free and open-source operating system that was created by Linus Torvalds in 1991. It is based on the Unix operating system and is used on a wide range of devices, including servers, desktops, laptops, and mobile phones. One of the main features of Linux is its flexibility and customization, as users can choose from a variety of different distributions (versions) and can also modify the source code to suit their specific needs. Linux is known for its stability, security, and performance, making it a popular choice for many businesses and organizations. It is also widely used in the development of software and applications, as it provides access to a wide range of programming languages and tools.

Now, What is a Shell?

A shell is a command-line interface for interacting with an operating system. It allows users to enter commands, which the shell then executes and returns the output. The shell is an important component of an operating system, as it provides a way for users to interact with the system and execute commands to perform various tasks. There are many different types of shells available, each with their own set of features and capabilities. Some common shells include the Bourne shell (sh), the C shell (csh), and the Bourne-Again shell (bash). Bash is the most widely used shell on Linux and macOS systems, and it is also available on Windows through the Windows Subsystem for Linux.

Let's see some of the most used commands in LINUX.

pwd

The pwd command stands for "print working directory". It is used to display the current working directory (i.e., the directory that you are currently in) in the terminal.

Here is an example of how to use the pwd command:

pwd
/home/user/documents

In this example, the output of the pwd command is /home/user/documents, which indicates that the current working directory is the "documents" directory located in the "user" directory in the root directory (i.e., /).

You can use the pwd command at any time to see the full path of the current working directory. This can be useful when you are navigating through different directories and want to know exactly where you are in the file system.

ls

The ls command is used to list the files and directories in a directory. By default, it will show the names of the files and directories in the current working directory.

Here is an example of how to use the ls command:

ls
file1.txt file2.pdf directory1/

In this example, the ls command lists three items in the current directory: file1.txt, file2.pdf, and directory1/. The / after directory1 indicates that it is a directory, rather than a file.

You can use various options with the ls command to modify its behavior. For example, you can use the -l option to display the files and directories in a long format, which includes additional information such as the file size and permissions. You can also use the -a option to show hidden files (i.e., files that are normally not displayed because their names start with a dot).

Here is an example of using the ls command with the -l and -a options:

ls -la
total 16
drwxrwxr-x 2 user user 4096 Jan 7 16:22 .
drwxrwxr-x 18 user user 4096 Jan 7 16:22 ..
-rw-rw-r-- 1 user user 10 Jan 7 16:22 .hidden_file
-rw-rw-r-- 1 user user 34 Jan 7 16:22 file1.txt
-rw-rw-r-- 1 user user 114 Jan 7 16:22 file2.pdf
drwxrwxr-x 2 user user 4096 Jan 7 16:22 directory1/

In this example, the ls command lists all files and directories in the current directory, including hidden files. The long format output includes from left to right:

  • the file permissions (and if your system supports ACLs, you get an ACL flag as well)
  • the number of links to that file
  • the owner of the file
  • the group of the file
  • the file size in bytes
  • the file modified datetime
  • the file name

cd

The cd command stands for "change directory". It is used to change the current working directory in the terminal.

Here is an example of how to use the cd command:

cd /home/user/documents

In this example, the cd command changes the current working directory to the "documents" directory located in the "user" directory in the root directory (i.e., /).

You can also use the cd command to navigate to directories relative to the current working directory. For example, you can use cd .. to move up one directory level, or you can use cd directory1 to move into a subdirectory called "directory1".

mkdir

The mkdir command stands for "make directory". It is used to create a new directory in the file system.

Here is an example of how to use the `mkdir` command:

mkdir new_directory

In this example, the mkdir command creates a new directory called "new_directory" in the current working directory.

You can also use the mkdir command to create multiple directories at once by specifying a list of directory names separated by spaces. For example:

mkdir dir1 dir2 dir3

You can use the -p option to create intermediate directories that do not yet exist. For example:

mkdir -p path/to/new_directory

In this example, the mkdir command will create the "new_directory" directory inside the "path/to" directory, creating both directories if they do not already exist.

touch

The `touch` command is used to create a new file or update the timestamp of an existing file.

Here is an example of how to use the `touch` command:

touch newfile.txt

In this example, the touch command creates a new, empty file called "new_file.txt" in the current working directory.

You can also use the touch command to update the timestamp of an existing file. For example:

touch existing_file.txt

In this example, the touch command updates the timestamp of the "existing_file.txt" file to the current date and time.

ln

The ln command is used to create links between files in a file system. There are two types of links: soft links (also called symbolic links) and hard links.

  • A soft link (symbolic link) is a special type of file that contains a reference to another file or directory. When the soft link is accessed, it behaves as if it were the file or directory that it is linked to.

    Here is an example of how to create a soft link using the ln command:

    ln -s /path/to/original_file.txt soft_link.txt

    In this example, the ln command creates a soft link called "soft_link.txt" that points to the file "original_file.txt" located at /path/to/original_file.txt.

  • A hard link is a direct link to the physical data of a file, rather than a reference to the file itself. A hard link behaves just like the original file, and changes to the hard link are reflected in the original file and vice versa.

    Here is an example of how to create a hard link using the ln command:

    ln /path/to/original_file.txt hard_link.txt

    In this example, the ln command creates a hard link called "hard_link.txt" that points to the file "original_file.txt" located at /path/to/original_file.txt.

The ln command is a useful tool for creating links between files in a file system. Soft links allow you to create a reference to a file or directory without taking up additional space, while hard links allow you to access the same data from multiple locations without taking up additional space.
